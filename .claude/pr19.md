Ports the Krate schema from `legacy/db.sql` to Laravel migrations + seeders. Part of #27.

## Migrations
- **Reshaped** the default users migration to Krate's shape: `first_name`, `last_name`, `username` (unique), `email` (unique), `password`, `role`, `email_verified_at`, `remember_token`, timestamps. Kept Laravel's `password_reset_tokens` + `sessions` tables (Breeze/password reset use them).
- **`settings`** — key/value store (`setting_key` unique, type/category/description/is_private, `created_by`/`updated_by` FKs).
- **`vinyl_records`** — the core catalog with the original enums (`format`/`speed`/`condition`). **Shared/global catalog — no `user_id`** (per-member ownership is deferred to #29).
- Laravel-idiomatic `id` primary keys (legacy `record_id`/`user_id`/`setting_id`).

## Seeders (data ported verbatim from legacy/db.sql)
- `UserSeeder` — 6 users; **bcrypt hashes preserved verbatim** (no re-hash → existing logins keep working).
- `SettingSeeder` — 16 settings.
- `VinylRecordSeeder` — 5 records.

Only the three real Krate tables are migrated; the legacy CMS cruft (`admins`, `contact_list`, `pages`, `rankings`, `subjects`) is intentionally left behind.

## Database
- Uses a **dedicated `krate` database** so the legacy `db` (CMS-era tables) stays intact as a reference. Provisioned automatically via a DDEV `post-start` hook (`.ddev/config.krate.yaml`), so `ddev start` + `php artisan migrate --seed` works on a fresh checkout.
- **Restored `database` session/cache drivers**, replacing the file-based interim from #18.

## Notes
- "No more runtime `ensureTableExists()`" is satisfied by design — that legacy behavior lives only in `legacy/` and is not carried into the Laravel app.
- Some legacy `notes` text carries pre-existing mojibake (the legacy `vinyl_records` table was `latin1`); preserved as-is, encoding cleanup is out of scope here.

## Verification
```
ddev artisan migrate:fresh --seed   # all green
# users=6, settings=16, vinyl_records=5; app boots HTTP 200
```

Closes #19.
